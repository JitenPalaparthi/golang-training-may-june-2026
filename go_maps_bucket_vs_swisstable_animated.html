Teaching Summary
Bucket-based map:
A hash chooses a bucket. Each bucket has 8 slots. Each slot stores key/value data and tophash metadata. If many keys land in the same bucket, an overflow bucket is linked.
Swiss-table-style map:
A hash is split. One part guides table/group probing. A smaller fingerprint is stored in control bytes. Lookup checks control bytes first, then compares keys only for possible matches.
