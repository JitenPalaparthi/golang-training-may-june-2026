Animated Steps
Bucket count = 2B   |   Each bucket has 8 slots   |   Overflow bucket is used when many keys land in the same bucket
Teaching Summary: In the older bucket-based Go map design, the runtime keeps an hmap header. The hash of a key selects a bucket from the bucket array. Each bucket has 8 slots. Every slot stores a key/value pair, and the bucket also keeps tophash metadata to quickly reject non-matching slots. If too many keys land in the same bucket, Go links an overflow bucket. During growth, Go gradually moves entries from old buckets to new buckets.
